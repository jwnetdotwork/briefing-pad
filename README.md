# BriefingPad

BriefingPad は、`生活と奉仕の集会` の実演準備を支援する macOS アプリです。

Notion で管理している原稿を取り込み、パートごとに録音・文字起こし・分析を行い、観察メモや良かった点候補、AI メモを整理します。パート終了時には、生成した AI メモを Notion に書き戻せます。

## 主な用途

- Notion に置いた集会用の原稿をアプリへ取り込む
- パートごとに録音して文字起こしを蓄積する
- 文字起こしから観察メモと良かった点候補を自動で拾う
- パート終了後に、短評用の AI メモを生成する
- 生成した AI メモを Notion に同期する

## 対応環境

- macOS 15.5 以降
- 日本語の音声認識が利用できる環境
- OpenAI API Key
- Notionコネクト アクセストークン

## 事前準備

### 1. OpenAI API Key を用意する

アプリの文字起こし分析と AI メモ生成で OpenAI API を使います。

### 2. Notion インテグレーションを作成する

Notion の [開発者ポータル](https://app.notion.com/developers/connections) から新規コネクトを作成し、インテグレーショントークン(アクセストークン)を取得します。

### 3. 対象ページを共有する
設定→コネクト→管理→コネクトとトークンを管理する→すべてのコネクト→対象のコネクトの「…」→コネクトを管理を開きます｡  
ページアクセスを管理で、Notion の対象ページもしくは親ページにチェックを入れてコネクトがアクセスできるようにします｡  

共有が足りない場合は、インポート時に権限エラーになります。

## 初期設定

1. アプリ右上の `設定` を開く
2. `OpenAI API Key` を保存する
3. `Notion インテグレーション・トークン` を保存する
4. `保存` を押す

保存した情報はローカルの Keychain に保存されます。

## Notion 連携の手順

1. アプリ右上の `Notionインポート` を開く
2. Notion ページの URL かページ ID を貼り付ける
3. `プレビュー確認` を押す
4. セッション名とパート一覧、未解釈ブロック数を確認する
5. 問題なければ `インポート確定` を押す

URL は `https://app.notion.com/...` 形式でも、ページ ID 直指定でも使えます。

## Notion ページの構成

取り込みは、Notion ページのブロック構成にかなり依存します。

特に、章見出しとパート見出しは次の形に揃えるのが安定です。

```md
神の言葉の宝          <- heading_2
  3. パート名           <- heading_3
    （4分）場面や資料
    📓 学習ポイント
    ・...
    👀 観察メモ
    ・...
    👍 どこがどのように良かったか
    ・...
    🤖 AIメモ

野外奉仕に励む        <- heading_2
  4. パート名           <- heading_3
    （5分）
    ...
```

### 認識される見出し

- `heading_2` の章見出し
- `heading_3` のパート見出し

### 章見出し

次の 2 つの章名をそのまま使ってください。

- `神の言葉の宝`
- `野外奉仕に励む`

この 2 章の外にあるブロックは、原則として取り込み対象外です。

### パート見出し

パート見出しは `1.` `2.` `3.` のように、数字 + `.` で始めてください。

例:

- `3. 聖書朗読 山田二郎`
- `4. 会話を始める 山田花子/山田花枝`

### 取り込みやすい配置

- パート見出しの直後に、時間と、設定や場面を置く
- 各セクション見出しは、1 ブロック 1 行にする
- 各項目は、1 ブロックにつき 1 件にする
- セクション見出しの文言は毎回同じにする

## 使う絵文字ラベル

現行の取り込みで安定して使えるラベルは次の通りです。

- `📓 学習ポイント`
- `👀 観察メモ`
- `👍 どこがどのように良かったか`
- `🤖 AIメモ`

### ラベル運用のコツ

- 絵文字と文言は毎回同じにする
- ラベルは単独ブロックに置く
- ラベルの下には、箇条書きか短い段落で項目を並べる
- 1 ブロックに複数のラベルを混ぜない
- ラベル名を勝手に言い換えない
- `🤖 AIメモ` はアプリ画面上の表記です

### 使わない方がよいラベル

次のラベルはコード上に予約がありますが、現時点では自動取り込みの主対象ではありません。

- `☔ 次の一歩`
- `👪 まとめ`
- `事前情報`

`事前情報` は補足欄として置いておくのは構いませんが、構造化された項目としては扱われません。

## AI メモの書き込み先

AI メモは、アプリ上では `🤖 AIメモ` セクションに表示されます。

アプリでは、パート終了後または手動の再生成時に AI メモを作成し、該当ブロックへ同期します。

Notion への最終出力では `🤖 コメント素材` として書き込まれるため、テンプレート側は `🤖 AIメモ` でも `🤖 コメント素材` でも運用できます。表記を統一したい場合は、アプリ画面の `🤖 AIメモ` に合わせるのが分かりやすいです。

## 画面の見方

### 上部バー

- セッション選択
- Notion インポート
- 設定
- 削除メニュー

### パート一覧

現在のセッションに含まれるパートを横並びで表示します。

左右の矢印で前後のパートに移動できます。

### 録音と再生

- `開始` で録音を始める
- `停止` で録音を止める
- `再生` で保存済み音声を再生する
- `再生停止` で再生を止める
- `パート終了` でそのパートを確定する

### 文字起こし

録音中の発話がここに表示されます。

### 観察メモ / 良かった点候補

文字起こしの内容に応じて、自動で候補が表示されます。

### AI メモ

ここに、短評用のコメント素材が表示されます。

- パート終了時に自動生成される
- 手動で `再生成` できる
- Notion 連携が有効なら、対応する `🤖 AIメモ` ブロックにも反映される

## うまく取り込めないとき

- プレビューの未解釈ブロック数が多い場合は、見出し階層を見直す
- 章名が完全一致しているか確認する
- パート見出しが `3.` のように数字 + `.` で始まっているか確認する
- Notion ページがインテグレーションに共有されているか確認する
- 設定画面で OpenAI API Key と Notion トークンが保存されているか確認する

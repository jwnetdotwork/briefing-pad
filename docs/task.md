# Phase 8：Notion読み込み

ここからNotion連携。

ただし、ここでの Notion は **セッション初期化時のパート情報インポート元** として扱う。

- Notion は読み込み専用
- セッション進行中の細かい状態はローカル保存
- 再保存や双方向同期はしない
- 1回のセッション開始時にテンプレートを取り込む

つまり、

```text
Notion = パート情報の土台
ローカル = 実演中の状態・文字起こし・LLM結果・確定済みAIメモ
```

として分ける。

## 8-1. Notion APIクライアント作成

- [ ] Integration Token設定
- [ ] Page ID入力欄
- [ ] ページブロック取得
- [ ] 子ブロック再帰取得

## 8-2. Notionページ指定

- [ ] Notion URL貼り付け
- [ ] URLからPage ID抽出
- [ ] 接続テスト
- [ ] 読み込み結果表示

## 8-3. ルールベースパーサー実装

- [ ] 1ページ = 1セッションテンプレートとして扱う
- [ ] パート見出しは `Part {番号}. {タイトル}` 系を検出する
- [ ] パート内の `持ち時間` を検出する
- [ ] パート内の `場面` を検出する
- [ ] `📓学習ポイント` を検出する
- [ ] `👀観察メモ` を検出する
- [ ] `👍どこがどのように良かったか` を検出する
- [ ] `🤖AIメモ` を検出する
- [ ] 各ブロックIDを保持する
- [ ] 既知ラベル以外のブロックは raw として残す
- [ ] 取り込み後にローカル用の `PartDefinition` に変換する

パース対象の最小単位は次でよい。

```text
Part 1. 挨拶をする
持ち時間: 2分
場面: 相手に自然に声をかける導入

📓学習ポイント
- 笑顔で最初の一言を置く
- 相手が返しやすい質問を混ぜる

👀観察メモ
- 笑顔で挨拶できているか
- 相手の反応を待てているか

👍どこがどのように良かったか
- 自然な笑顔で話しかけられていた
- 最初の一言が落ち着いていた

🤖AIメモ
...
```

このフェーズで取るのは、あくまで初期化に必要な要素だけ。

- `number`
- `title`
- `durationMinutes`
- `setting`
- `learningPoints`
- `observationItems`
- `positiveItems`
- `aiMemoBlockId`
- `rawMarkdown` もしくは同等の元テキスト

セッション固有の以下は Notion に置かない。

- 文字起こし
- チャンク
- LLM結果
- 実演中の状態
- 終了後の確定まとめ

## 8-4. パース結果確認画面

- [ ] 検出されたパート一覧表示
- [ ] パート番号・タイトル・持ち時間・場面を表示
- [ ] 学習ポイント数表示

最初の実装では **Notionを後回し** にした方がいい。  
でもNotion前提の設計はしておく。

つまり、

```text
PartDefinitionの取得元を差し替え可能にする
```

だけ最初から意識する。

```text
最初：LocalJSONPartProvider
後で：NotionPartProvider
```

みたいにしておくと、後からNotion連携しても大きく崩れない。
